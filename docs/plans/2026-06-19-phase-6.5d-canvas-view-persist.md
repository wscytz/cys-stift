## 1. 范围

### ✅ 本阶段做

#### 1.1 视图存储 web-local
- 新建 `apps/web/src/lib/canvas-view-store.ts`:
  - localStorage key `cys-stift.canvas-view.v1`
  - 单 canvas(MVP),不分 canvasId
  - 类型:
    ```ts
    export interface CanvasView {
      zoom: number
      panX: number
      panY: number
      gridMode: 'snap' | 'free'
      gridSize: number
    }
    ```
  - API:
    ```ts
    export const canvasViewStore = {
      get(): CanvasView
      update(patch: Partial<CanvasView>): void
    }
    export function useCanvasView(): { view: CanvasView; ready: boolean }
    ```
- 默认 `{zoom:1, panX:0, panY:0, gridMode:'snap', gridSize:8}` (同 Phase 4 canvas-editor onMount 默认)
- snapshot 引用稳定(同 draft-store 模式)

#### 1.2 canvas-editor 加载 + 持久化
- `apps/web/src/features/canvas/canvas-editor.tsx`:
  - onMount:先 `canvasViewStore.get()` → `editor.setCamera({x: panX, y: panY, z: zoom})` + `editor.updateInstanceState({isGridMode: gridMode === 'snap'})` + `editor.updateDocumentSettings({gridSize})`
  - 监听变化:`editor.store.listen` 监听 camera + isGridMode 变化,防抖 500ms → `canvasViewStore.update(...)`
  - 不监听 gridSize(用户不能改)
- 删除 `editor.updateInstanceState({isGridMode: true})` 和 `editor.updateDocumentSettings({gridSize: 8})` 硬编码(用 store 的值)

### ❌ 本阶段不做

- 多画布 view(只 DEFAULT 一个 canvas)→ spec §4.9 schema 留位,UI 留后
- viewJson 进 `canvases` SQLite 表 → Phase 8 替换 Tauri fs 时再做
- domain `CanvasService.updateView` → 留 P6.5+(需 CanvasRepository 实现,当前 web-local 简化为 lib/canvas-view-store)
- 视图 history(回到上次位置)→ 留后
- tldraw 自带 store 持久化 → 不沿用(我们的 card position 走 §6.11 防抖回写,view 同样思路)

## 6. 风险

| 风险 | 处理 |
|---|---|
| tldraw camera 监听触发过于频繁(每次 pan 1px) | 防抖 500ms,同 canvas-binding 防抖 300ms 思路 |
| onMount 时 store 未 hydrate | localStorage 同步读(get() 同步),无需 ready flag |
| isGridMode + user.isSnapMode 同步 | 沿用 Phase 5 closeout 决策 |
| 移动端 pan 性能 | tldraw 自带节流;防抖 500ms 兜底 |
| gridSize 8 不能改 | 不监听;spec §4.3 锁死 |
| localStorage 损坏 / 旧版本格式 | try/catch + 降级到默认值 |
