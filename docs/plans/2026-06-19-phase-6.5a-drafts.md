## 0. 目标

兑现 spec §5.5 "输入即保存草稿":Mini Input 输入标题/body → 防抖 500ms 自动存草稿 → 关闭 Mini Input → 重新触发 → **草稿恢复**,标题/body 还在;提交成功后草稿清除。

**核心承诺**:Mini Input 输入 "灵感 A" → 关闭 → 重开 → 看到 "灵感 A" 还在 → 改成 "灵感 B" → 关闭 → 重开 → "灵感 B"(最新)→ Cmd+Enter 保存成功 → 重开 → **空草稿**(已提交清除)。

## 1. 范围

### ✅ 本阶段做

#### 1.1 草稿存储层(`apps/web/src/lib/draft-store.ts`,web-local)
- 独立 localStorage key:`cys-stift.drafts.v1`
- 单例 in-memory store + `useSyncExternalStore`(与 db-client 同模式,保证 snapshot 稳定)
- API:
  ```ts
  export type DraftKind = 'capture' | 'manual'
  export interface Draft {
    kind: DraftKind
    title: string
    body: string
    updatedAt: Date
  }
  export const draftStore = {
    get(kind: DraftKind): Draft | null
    upsert(kind: DraftKind, patch: { title: string; body: string }): void
    clear(kind: DraftKind): void
    subscribe(cb: () => void): () => void
  }
  export function useDraft(kind: DraftKind): { draft: Draft | null; ready: boolean }
  ```
- **不进 domain**:草稿是 web-local UI 状态,不是核心业务实体(domain 零依赖 + 不污染核心数据模型)。spec §5.5 说 "SQLite + 本地状态",但浏览器端当前是 localStorage(Phase 2 决策),草稿跟 cards 走同样的 web-local 存储;Phase 8 Tauri 端草稿跟 Tauri fs 走(留后)。

#### 1.2 Mini Input 接草稿
- `apps/web/src/features/capture/mini-input.tsx`:
  - 初次渲染:读 `draftStore.get('capture')` 初始化 title/body
  - title/body 变化:防抖 500ms → `draftStore.upsert('capture', {title, body})`
  - 提交成功后:`draftStore.clear('capture')` + onClose
  - Escape 关闭:**不清除**草稿(用户可能误关;spec §5.5 "输入即保存" 暗示关闭也应保留)

#### 1.3 inbox CreateCardForm 接草稿
- `apps/web/src/app/inbox/create-card-form.tsx`:
  - 同样读 `draftStore.get('manual')` 初始化
  - 任意字段(title/body/links/code/quotes)变化:防抖 500ms → `draftStore.upsert('manual', ...)`
  - 提交成功后:`draftStore.clear('manual')`
- **注意**:manual 草稿含多媒介,`Draft` 类型需扩展 payload(或 manual 用不同子类型)。决策:**`Draft` 的 payload 用 `unknown` JSON,各消费方自己 cast**;capture 只 title+body,manual 含完整 CreateCardInput。

#### 1.4 防抖 hook(`apps/web/src/lib/use-debounced-callback.ts`)
- 通用防抖,500ms,可 cleanup
- 避免 Mini Input / CreateCardForm 各写一遍

### ❌ 本阶段不做(明确留后)

- **Tauri fs 草稿落盘** → Phase 8(当前 web-local localStorage 够用)
- **草稿版本历史 / 多草稿** → 留后(spec §5.5 单数 "草稿")
- **草稿冲突解决**(同 kind 跨 tab 写)→ 留后(localStorage `storage` event 跨 tab 同步留后)
- **草稿手动清除按钮** → 留后(提交自动清;Escape 保留)
- **wa-sqlite 替换 localStorage** → Phase 2.5(独立 phase)

## 3. 任务清单

### P6.5a-T1 · `use-debounced-callback` hook
- 新建 `apps/web/src/lib/use-debounced-callback.ts`:
  ```ts
  export function useDebouncedCallback<T extends (...args: any[]) => void>(fn: T, delay: number): T
  ```
  - 返回一个稳定 ref 的函数,内部 setTimeout 延迟调用,新调用取消旧 timer
  - unmount 时 cleanup timer
- 0 依赖

### P6.5a-T2 · `draft-store.ts` 草稿存储层
- 新建 `apps/web/src/lib/draft-store.ts`(如 §1.1)
- 模块级单例:`_drafts: Map<DraftKind, Draft>` + `_subscribers`
- localStorage 序列化:`{ drafts: Record<DraftKind, Draft> }`,Date 走 ISO 字符串
- `useDraft(kind)` hook:返回 `{ draft, ready }`,ready 在 hydrate 后 true
- **snapshot 稳定性**:缓存 `Record<DraftKind, Draft>` 对象,只在 Map 变化时重分配(同 db-client 模式)

### P6.5a-T3 · Mini Input 接草稿
- `mini-input.tsx`:
  - `useState` title/body 初值从 `useDraft('capture').draft` 读(或空)
  - `useDebouncedCallback` 包 `draftStore.upsert('capture', {title, body})`,delay 500
  - title/body onChange → setLocal + 触发防抖 upsert
  - 提交成功(onSubmit 后)→ `draftStore.clear('capture')` 再 onClose
  - Escape → onClose **不清草稿**

### P6.5a-T4 · CreateCardForm 接草稿
- `create-card-form.tsx`:
  - 初值从 `useDraft('manual').draft` 读(cast payload 为 CreateCardInput shape)
  - 防抖 upsert 'manual' + 完整 payload(title/body/links/code/quotes)
  - 提交成功 → clear('manual')
- **注意**:`Draft.payload` 用 `unknown`,manual 消费方 cast;不污染 type 系统

## 6. 风险

| 风险 | 处理 |
|---|---|
| localStorage 满(quota)抛错 | draft-store 所有读写 try/catch,失败静默(草稿丢失不致命,用户输入仍在 React state)|
| 草稿 Date 序列化跨刷新还原 | ISO 字符串存,load 时 `new Date(str)` 还原;同 db-client 模式 |
| CreateCardForm 多媒介草稿 payload 复杂 | `Draft.payload: unknown`,manual 消费方 cast;不强制 domain 类型 |
| 跨 tab 草稿冲突 | MVP 不处理(localStorage `storage` event 跨 tab 同步留后);单 tab 用户为主 |
| 防抖与提交竞态(用户在防抖窗口内提交) | 提交成功后显式 `clear('capture')`,覆盖未 flush 的防抖;clear 是同步的 |
| 修改 tagged Phase 3 CreateCardForm 破坏多媒介 | 只加 useEffect 草稿读写 + 防抖 upsert;不动表单结构;回归 p3-shots 确认 |
| Mini Input 早返 null 后草稿初值错过 autoFocus | `useDraft` ready 后才渲染;或在 ready 前用空值,ready 后用 useEffect 填回 |
