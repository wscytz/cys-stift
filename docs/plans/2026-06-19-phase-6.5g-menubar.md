## 0. 目标

顶部全局菜单栏(不依赖 tldraw chrome,自建)+ CaptureSinkRegistry 抽象支持多 sink 注册。Capture 入口走 MenuCaptureSink,`source.kind='menubar'`。

**核心承诺**:任意路由(包括 `/`、`/inbox`、`/canvas`、`/archive` 顶部都可见菜单栏 → 点 "Capture" → Mini Input 弹出 + `card.source.kind === 'menubar'`)。

---

## 1. 范围

### ✅ 本阶段做

#### 1.1 顶部全局菜单栏
- 新建 `apps/web/src/components/app-menu.tsx`(或 `features/menu/app-menu.tsx`):
  - 横向链接条:Inbox / Canvas / Archive / Capture
  - 4 入口:与首页 home__nav-link 风格一致(更紧凑)
  - 高亮当前路由(`usePathname()`)
  - 顶部 8px 蓝条(与 home__capture 区分:菜单栏中性,蓝条表示 system region,spec §5.2)

#### 1.2 CaptureSinkRegistry
- 改 `apps/web/src/features/capture/capture-sink.ts`:
  - 加 `register(kind: CaptureSource['kind'], sink: CaptureSink)` + `submit(input)` 按 source.kind 路由
  - 当前只注册 web 1 个(Phase 6)
  - CaptureHost.submit 用 `registry.submit` 而非 `new WebCaptureSink(service)`

#### 1.3 MenuCaptureSink
- 新建 `apps/web/src/features/capture/menu-capture-sink.ts`:
  - `class MenuCaptureSink implements CaptureSink`:与 WebCaptureSink 等价,但 `source.kind = 'menubar'`
  - (实际实现可在 `capture-sink.ts` 内部提供 factory,不必单独文件;**决策**:`menu-capture-sink.ts` 独立文件,与 WebCaptureSink 对称)

#### 1.4 menu 入口触发 Mini Input
- `AppMenu` "Capture" 按钮:dispatch CustomEvent('open-capture') → CaptureHost 监听 → setOpen(true)
- 或更简单:用全局 Zustand-like store 共享 open 状态
- **决策**:用 **CustomEvent** 不引入新依赖,Phase 6 closeout 已知/后续 "Capture 入口纯展示" → 升级到 dispatch event

### ❌ 本阶段不做

- 真实 OS 菜单栏(Tauri menubar)→ Phase 8
- 移动端 → 留后
- Webhook / mobile / alfred sink → 留后
- 菜单栏用户自定义 → P6.5h
