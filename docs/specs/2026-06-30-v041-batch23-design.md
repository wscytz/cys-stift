## 批 2 — P1 交互

### #6. resize 不遵守 snap/grid

**问题**:snap 模式下拖卡位置吸附,但拖角 resize 尺寸不是 8 倍数,手感不一致。

**根因**:`self-built-adapter.ts:759-768` onMove resize 分支拿 `resizeGeometry(handle, start, p)` 的结果直用,`p` 未经 snapCoord。`self-built-resize.ts` resizeGeometry 末尾仅 Math.round。

**设计**:onMove resize 分支,拖动角点 `p` 先 snap(snap 模式)再传 resizeGeometry。对角由 `start` 固定(resizeGeometry 已保证),snap 拖动角即让尺寸落网格。

```ts
// onMove resize 分支(约 :763)
const g = resizeGeometry(this.resizing.handle, this.resizing.start, p)
```
改为:
```ts
const sp = this.view.gridMode === 'snap'
  ? { x: this.snapCoord(p.x), y: this.snapCoord(p.y) }
  : p
const g = resizeGeometry(this.resizing.handle, this.resizing.start, sp)
```

`snapCoord`(:1150,私有)复用 drag 已有的;`this.view.gridMode` 现有字段。

**验证**:snap 模式拖角 resize,松手后 w/h 是 8 倍数;free 模式不变(回归)。

### #7. RelationPanel 静默自动改箭头样式

**问题**:选中无标签箭头,inferRelationTypeFromContext 自动 applyRelationType 改颜色/线型/标签,无提示,用户以为 bug。

**根因**:`relation-panel.tsx:93-104` useEffect auto-apply 后无反馈。

**设计**:保留自动识别(智能特性),auto-apply 后弹一次 toast「已识别为 X,可点其他类型覆盖」。键控(`appliedKey`,每 id:text:type 一次)不会刷屏。

```ts
// relation-panel.tsx useEffect auto-apply 内,applyRelationType 之后加:
pushToast(t('relation.autoApplied', { type: inferred.label }), 'info')
```

需 i18n 加 `relation.autoApplied`(带 type 插值)+ 复用现有 pushToast。确认 relation-panel 已能访问 pushToast(若无需 import + i18n hook 已有)。

**验证**:选中无标签、可推断的箭头 → toast 出现一次;再选同箭头不重复;切换类型后重选不刷屏。

### #8. 纯点击/点空白产生空 undo 步

**问题**:select 点选(没拖)/ eraser 点空白(没擦到)→ onDown 已 pushUndo,undo 栈污染。

**根因**:
- select onDown(`:698-700`):设 dragGroup 后无条件 `pushUndo + coalescing=true`
- eraser onDown(`:566-577`):先 `pushUndo + coalescing=true` 再 `eraseAt(p)`,eraseAt 返回 null(空白)时已推

**设计**:lazy pushUndo —— 实际操作发生才推。

**select drag**:onDown 设 dragGroup 时**不** pushUndo(coalescing 保持 false);onMove drag 分支首次进入时 `if (!this.coalescing) { this.pushUndo(); this.coalescing = true }` 再移动。纯点击不进 move → 无 undo 步。

```ts
// onDown select 分支(约 :698-700),删掉:
//   this.pushUndo()
//   this.coalescing = true
// 改为只设 dragGroup(pushUndo 移到 onMove 首次)
```

```ts
// onMove drag 分支(约 :775-785),循环前加:
if (!this.coalescing) {
  this.pushUndo()
  this.coalescing = true
}
```

**eraser**:onDown 重排 —— 先 eraseAt,命中了才 pushUndo+coalescing。

```ts
// onDown eraser 分支(约 :566-577),把:
//   this.pushUndo()
//   this.coalescing = true
//   const id = this.eraseAt(p)
//   this.erasing = { lastId: id }
// 改为:
const id = this.eraseAt(p)
if (id) {
  this.pushUndo()
  this.coalescing = true
}
this.erasing = { lastId: id }
```

onMove erasing 分支:后续命中被 coalescing 抑制(若首擦命中已 set);若首擦没命中(点空白没 set coalescing),后续 move 首次命中需 lazy pushUndo。在 onMove erasing 分支 eraseAt 命中前加 `if (id && !this.coalescing) { this.pushUndo(); this.coalescing = true }`。

**边界**:
- onUp 原本 `coalescing = false`(若从未 true 无害)。
- drag 首次 move lazy pushUndo 后,后续 move coalescing=true 不重复推 —— 与原 coalescing 语义一致。
- multi-drag(多选拖)首个 move 推一次,整个拖拽 1 步 undo,不变。

**验证**:点选卡(不拖)→ undo 栈不增;eraser 点空白 → undo 栈不增;正常拖/擦 → 1 步 undo(回归)。

## 批 3 — P1 UI

### #9. error.tsx 全量硬编码中文

**根因**:`apps/web/src/app/error.tsx:30-48` 崩溃页文案全中文,未走 t()。组件是 'use client',useI18n 可用。

**设计**:import useI18n;文案移到 `lib/i18n/messages.ts` 新增 `error.boundary.*` 键(title/subtitle/retry/home/errorCode/tech/techHint)。

### #10. inbox/markdown.tsx 嵌入提示硬编码中文

**根因**:`apps/web/src/app/inbox/markdown.tsx:99-102` 三处嵌入错误提示中文。组件已接 useI18n(同文件别处用 t())。

**设计**:新增 `md.embed.cycleDepth`/`cycleRef`/`missing` 键(带 title 插值),用 t() 替换。

### #11. agent-confirm-card.tsx hex 写死

**根因**:`features/ai/agent-confirm-card.tsx:231` `#0a0a0a`/`#d40000`/`#6b6b6b` + `:232` `rgba(10,10,10,0.08)`。暗色下 #0a0a0a 描边在 #0a0a0a 底不可见 + 违反铁律。

**设计**:描边色改 `readToken('--color-black'/'--color-red'/'--color-gray', fallback)`(从 `@cys-stift/canvas-engine` import readToken,与 graph-canvas.tsx 同模式)。填充 `rgba(10,10,10,0.08)` → `readToken('--color-black-soft', …)` 或保留(填充色暗色影响小,主要修描边)。

### #12. tag-cloud 计数色 rgba 暗色不可读

**根因**:`features/tags/tag-cloud.tsx:153` `color: rgba(0,0,0,0.7)`。暗色 tag 深底 + 黑字对比度极低。

**设计**:改 `var(--color-black-soft)`(暗色自动反转为浅色)。

### #13. toolbar 标签字号 + disabled 对比度

**根因**:`app/canvas/page.tsx:1709/1715/1718` 字号 `10px`/`9px`(不进 token 阶,最小 xs=12px);disabled `opacity:0.55`。

**设计**:字号 `10px` → `var(--font-size-xs)`,`9px` → `var(--font-size-xs)`(统一,子工具不再更小);disabled `opacity:0.55` 保留(disabled 态 WCAG 豁免,字号修好后可读性已改善)。
