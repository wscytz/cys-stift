## 0. 目标

Inbox 卡片详情 Modal 加 "Send to canvas" 按钮 → 调 `service.moveToCanvas(cardId, { canvasId: DEFAULT, x, y, w, h, z })` → 卡出现在 `/canvas`(已绑定的 Card ShapeUtil 自动渲染)。

**核心承诺**:Inbox 卡片详情 → 点 "Send to canvas" → 卡出现在画布(浏览器刷新仍存在)+ 详情 Modal 显示 "On canvas" badge + 按钮变成 "Open on canvas" 跳画布。

## 1. 范围

### ✅ 本阶段做

#### 1.1 详情 Modal "Send to canvas" 按钮
- `apps/web/src/app/inbox/page.tsx` 的 `CardDetail`:
  - view mode 按钮组加 "Send to canvas"(primary variant,只在 `!card.canvasPosition` 时显示)
  - 已发送状态:按钮变 "On canvas"(disabled,带 "on canvas" badge 或同等视觉)
- onClick:调 `service.moveToCanvas(card.id, { canvasId: DEFAULT_CANVAS_ID, x: 100, y: 100, w: 200, h: 80, z: nextZ })`

#### 1.2 视觉契约
- 按钮样式沿用 inbox 风格(ghost / primary variant)
- "On canvas" 状态:按钮 disabled + 文案改 + 加绿/蓝 tag(沿用 archive region 蓝色系)
- 0 新依赖

#### 1.3 Canvas dblclick 路径(留轻触)
- 现有 Phase 4 实现的"画布双击空白建卡"路径不动(spec §6.3)
- 新路径"inbox→canvas send"是另一入口,不冲突

### ❌ 本阶段不做

- 画布"卡片删除时同时从 inbox 删"的级联 → 留后(Phase 4 软删已实现,卡软删后两视图都消失)

- Canvas dblclick 路径入 inbox(已在画布的卡重新"回 inbox")→ 留后
- 自动定位到画布当前空白区(智能 x/y)→ 留后;MVP 用固定位置 100,100 + z=max+1
