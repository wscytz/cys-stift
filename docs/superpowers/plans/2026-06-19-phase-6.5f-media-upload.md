## 0. 目标

用户能给卡片附加图片:`<input type=file>` → 读 ArrayBuffer → base64 → 存 `media-assets` localStorage key + card.media 数组加 MediaRef → 详情 Modal view 显示图片。

**核心承诺**:详情 Modal view 显示附加的图片 + 跨刷新保留。

---

## 1. 范围

### ✅ 本阶段做

#### 1.1 web-local media store
- 新建 `apps/web/src/lib/media-store.ts`:
  - localStorage key `cys-stift.media.v1`(独立)
  - 类型:`Record<MediaAssetId, { id, kind, mimeType, dataUrl: string, createdAt }>`
  - API:`attach(file: File): Promise<MediaRef>` / `getAsset(id): MediaAssetData | null` / `remove(id)`
  - 限制:文件 > 500KB 警告(仍允许,但提示用户)

#### 1.2 详情 Modal edit mode 加图片上传
- inbox/page.tsx `CardDetail` edit mode:
  - 加 `<input type=file accept="image/*" multiple>`
  - onChange → for each file → `mediaStore.attach(file)` → 累加到 `media` state
  - 显示已附加图片缩略图 + × 删除
- onSave 扩 patch 含 `media`(已 `Card.media: MediaRef[]` 字段)

#### 1.3 详情 Modal view mode 显示图片
- view mode 渲染 `card.media`:`<img src={dataUrl} />` 从 mediaStore.getAsset

### ❌ 本阶段不做(明确留后)

- OPFS 真实落盘(spec §4.5 Web=OPFS)→ Phase 2.5
- Tauri fs 落盘 → Phase 8
- 图片编辑(裁剪/旋转)→ 留后
- 拖放上传 → 留后
- OG 图片抓取 → 留后(spec §12 风险 #11)
- 大文件流式上传 → 留后
