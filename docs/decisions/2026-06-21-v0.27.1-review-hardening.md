# 2026-06-21 · v0.27.1-review-hardening

> 来源:大规模代码复审(grep 5 个 agent 分别审 domain/db/canvas/web/arch,发现 18 个问题,全部在本档关闭或明确 defer)。v0.26.4 bugfix 和 v0.27.0 canvas-m1 之后的"工程底盘"轮次。

## 发现与修复

### L1 — 数据丢失(3 项)

**1. 导入后内存状态不同步**
`db-client.ts` + `settings/page.tsx`。导入成功 → 800ms reload 窗口 → 任何操作(捕获/编辑/pin)触发 persist() → 用旧内存覆盖刚导入数据。
**修**:`db-client.ts` 新增 `rehydrateCards()`,导入成功后立即调,同步内存与磁盘。reload 保留为 belt-and-suspenders(其他 store 需要)。

**2. 跨 tab Date 字段存为 string**
`db-client.ts:73`。storage 事件直接 `_cards = parsed.cards`,不重建 `capturedAt`/`createdAt` 等 Date → 跨 tab 后 `card.capturedAt.toISOString()` 抛 `TypeError`,渲染崩溃。
**修**:抽取 `parseCardsRaw(raw)` + `reviveDate(v)` 函数,初始 load 和 storage handler 共享同一套 Date 重建逻辑。单张卡日期损坏不会抛弃整个列表。

**3. syncCardsToEditor 仅同步存在性,不同步几何(B3-bis)**
`canvas-binding.ts:200`。跨 tab storage 事件触发 sync → 已存在的 shape 被 skip → tab B 永远显示旧坐标。
**修**:`syncCardsToEditor` 的"Add missing"循环改为 drift 检测 + `updateShape` reconcile,与 `loadCardsIntoEditor` 一致。

### L1 — M1 label(1 项,自身引入)

**4. 关系箭头无可见标签,注释写错**
`relation-types.ts:applyRelationType`。我写 M1 时试 `richText` prop 失败(`Unexpected property`),就放弃了 label,并留注释说"3.15 的 richText 只能 create 时写"。**现场验证这是错的**:3.15.6 的 arrow prop 实际叫 `text: string`,`updateShape({ props: {text: 'Blocks'} })` 成功。我用错了字段名。
**修**:改为 `text: label`,删除错误注释,补上正确说明。

### L1 — import XSS 加固(3 项)

**5. links[].url 直出 `<a href>`**
`card-detail.tsx:268` 的 links 区绕过 markdown 的 `safeHref` 白名单,导入的 `javascript:` URL 可点击执行。
**修**:抽取共享 `safe-href.ts`(`safeHref()` + `isSafeImageDataUrl()`),card-detail 和 markdown 双消费。

**6. media dataUrl 无校验**
导入的 `mediaAssets` 不校验 MIME/大小/base64 格式,可注入任意大小的不可信编码。
**修**:`isSafeImageDataUrl` 只允许 `image/(png|jpeg|jpg|gif|webp);base64` + 大小上限 5MB。SVG 排除(XSS 向量)。导入时静默剔除不合格的 asset。

**7. importFromJson 结构校验不全**
只校验 `id/title/body/createdAt/updatedAt`,不校验 `type` 白名单/数组形状/link URL。
**修**:加 `CARD_TYPES` 白名单、`isObjectArray` 数组校验、link URL 白名单。序列化和写入的返回路径统一用 `fail()` 辅助函数。

### L1 — canvas listener 无 cleanup(2 项合一)

**8. writeback + snapshot listener 挂在 onMount,无清除**
`canvas-editor.tsx:141,147`。两个订阅的返回值丢弃,赌 tldraw 自动 dispose。同时 `__canvasEditor` global 不清(B8)。其它两个 bridge 都迁到了 useEffect cleanup,这俩没迁,不一致。
**修**:新增 `EditorBindingBridge`(useEffect 模式),管理 writeback unsub + snapshot persist timer + `delete window.__canvasEditor`(B8)。onMount 只保留 one-shot 初始化和 `onEditorReady`。

### L2 — domain 类型安全(4 项)

**9. domain TS strict 从不验证**
`packages/domain/package.json:test` 只跑 `vitest run`(转译不 typecheck)。`tsc --noEmit` 在测试文件有 12 个错误:`workspaceId` 多余字段、`fetchedAt` 缺失、`pinned` 缺失、`noUncheckedIndexedAccess` 命中。
**修**:`test` 改 `tsc --noEmit && vitest run`。`search.test.ts` 重写(用 base spread + `toCardId` 替代 `as any`)。`card-service.test.ts:156` 索引访问加 `?`。

**10. softDelete 不幂等 + lifecycle 方法返回 void**
`card-service.ts:187`。重复调 `softDelete` 每次重盖 `deletedAt`,永远返回 void 不可区分成功/失败。archive/unarchive 同理。
**修**:所有 lifecycle 方法统一返回 `boolean`(true=存在且突变)。`softDelete` 加 `if (card.deletedAt) return false`,不再重盖。

**11. UpdateCardPatch.color: Card['color'] 耦合**
`card-service.ts:43,50`。用 `Card['color']` 索引访问类型,耦合输入 DTO 到实体 shape。
**修**:`import type { ColorToken }` 直接命名。

**12. ensureDefault 写悬空 canvas 引用**
`workspace-service.ts:34`。`canvasId = generateId()` 写进 workspace,但 caller 从不知道、从没建这个 canvas。任何 resolve `defaultCanvasId` → Canvas 拿到 null。
**修**:改签名 `ensureDefault(defaultCanvasId: CanvasId, name = '默认工作空间')`,canvas 由 caller 传入,不允许无 canvas 的 workspace。

### L2/3 — 配置与工程(5 项)

**修**:每个 dev 页面函数体首行加 `if (process.env.NODE_ENV === 'production') return null`,prerender 时直接返回空。

**15. apps/web 依赖无用 native + 死包**

**16. packages/db 的 broken export**
`package.json` 的 `exports: {"./persistence": "./src/persistence/index.ts"}` 指向不存在的目录。
**修**:删除该 export 条目。

**17. 硬编码 rgba / 无 scrim token**
`design/page.tsx:275` + `mini-input.tsx:196` 写 `rgba(10,10,10,0.5)`,违反 token 纪律。
**修**:`tokens.css` 加 `--color-scrim: rgba(10,10,10,0.5)`,两处改为 `var(--color-scrim)`。
