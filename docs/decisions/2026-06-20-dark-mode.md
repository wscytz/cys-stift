# Phase dark-mode · 暗色模式(2026-06-20)

> 承接 spec §5.6 "MVP 不做,预留 token 抽象,未来加"。
> Plan:无独立 plan 文件(实现规模适中,tokens 抽象早就在 spec §5.1 预留;设计决策在小决策表里逐项记下)。
> Tag:**v0.17.0-dark-mode**

## 背景

spec §5.1 + §5.2 设计 tokens 抽象**早为暗色模式预留** —— 6 原色集(`red / yellow / blue / black / white / gray`)+ Region 映射,所有 component 用 `var(--color-*)` 引用而非写死 hex。spec §5.6 明确说 "MVP 不做,预留 token 抽象,未来加"。

web 端 Phase 0 至今 CSS 只一个 `:root` 主题(light)。本次补 dark variant。

## 决策

| 决策 | 选项 | 选择 | 理由 |
|---|---|---|---|
| 入口位置 | 多处切换 + system 检测 / **单一 settings 选择 + system 默认 + inline head script** | **single source** | settings 已是 web 唯一主题切换点(快捷键也在 settings);system default 尊重 OS 偏好;inline head script 在 first paint 前 apply,**避免 light→dark flash**。 |
| 6 原色保留 | 引入第七色 / **保留 6 原色调亮度** | **保留** | spec §5.2 明确"用户可重映射 region → token,但**不能新增 token**(保持包豪斯原色集);dark 模式下 hue token 调亮(浅→深→浅)保 AA 对比度。 |
| CSS 选择器 | `[data-theme='dark']` / **`:root[data-theme='dark']`** | **`:root[data-theme]`** | specificity 0,1,1 > Tailwind v4 `@theme` 在 `:root` 注入的同名变量(0,0,1),确保 dark variant 真生效(试过前者 → 被 Tailwind reset 覆盖,specificity 不够)。 |
| theme 默认 | 'light' / **'system'** | **system** | 尊重 OS 偏好;新用户 / 老用户(settings 没存 theme 字段)按 OS 来;显式选 light/dark 才覆盖。 |
| dark 实现方式 | CSS class on body / **CSS attribute on `<html>`** | **attribute on `<html>`** | element 选择 `:root[data-theme='dark']` 直接匹配,无 class 列表维护成本;inline script 在 `<head>` 同步设 attribute,first paint 即生效。 |
| tldraw bg | 不动 / **sync `--tl-bg` on `<body>`** | **sync** | tldraw canvas 自己管理背景色;`applyTheme` 在 body 设 CSS variable 让 tldraw overrides.css 引用,确保 dark 覆盖 tldraw 表面。 |
| apply 策略 | React useEffect / **inline head script + ThemeBoot 接管 live** | **inline + ThemeBoot** | React useEffect 在 hydration 后才跑,有 flash;inline script 同步执行在 first paint 前;ThemeBoot 客户端 mount 后接管后续 settings 变化 + OS 变化。 |

## 关键工程决策(总结)

- **`:root[data-theme='dark']`** 而非 `[data-theme='dark']`:specificity 压过 Tailwind v4 `@theme` reset。
- **inline head script + ThemeBoot**:前者防 first-paint flash,后者管 live 变化。
- **6 原色保留,调亮度**:dark 下 red `#d40000 → #ff4d4d`(brighten 保 AA)。
- **theme = 'system' 默认**:尊重 OS 偏好,无侵入。
- **public `settingsStore.subscribe`**:统一 hooks + imperative 消费(theme.ts 不是 hook)。
- **sync `--tl-bg` on `<body>`**:tldraw 表面跟随 dark 主题。
- **单一 settings 切换点**:无多处分散,future theme 切换逻辑都汇聚 `/settings`。

## 显式留后(YAGNI)

- per-component dark overrides(目前全局 token swap 覆盖 6 原色,组件零改动)
- 暗色模式 scheduled 自动切换(根据时间)
- 高对比度模式 / 颜色矫正(`prefers-contrast` / `prefers-reduced-motion` 在 CSS 里加一行即可,无 UI)
- 用户自定义 region → color 映射 UI(spec §5.2 提"用户可重映射",MVP 不暴露 UI)
- tldraw 自定义 dot grid 在 dark 模式下的颜色(目前是 `var(--color-gray)`,已经跟 token 走)
