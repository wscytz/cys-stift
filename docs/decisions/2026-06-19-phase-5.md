## 关键工程决策

1. **`useState<Editor>` 替代 `useRef<Editor>`** — Phase 4 用 ref 留坑：ref 改值不触发 re-render，toolbar 按钮永远 disabled。Phase 5 第一个真依赖 editor 的功能（snap 切换）暴露。改 state 让 toolbar 跟着 mount 重渲染。这是 React 经典坑，Phase 4 acceptance 没断言 editor.handle → UI 联动所以没发现。
2. **toggle 同时设 `isGridMode` + `user.isSnapMode`** — tldraw v3 这俩**是独立的**：`isGridMode` 是 snap 总开关（DefaultCanvas / Pointing / Translating 都读），`isSnapMode` 只是 Ctrl 时的反转行为（snap on + ctrl = off；snap off + ctrl = on）。context7 文档没说，靠读 `Translating.ts` source 才发现。两者必须同步。
3. **gridSize 显式设 8** — tldraw v3 默认 `gridSize: 10`，spec §4.3 要 8。P5-T4 验证时读 `editor.getDocumentSettings().gridSize` 发现，加 `editor.updateDocumentSettings({ gridSize: 8 })`。
4. **缩放按钮用本地 `<button>` 而非 `@cys-stift/ui` Button** — Button 高 40px + padding 大不适合 47px 黑条内紧凑布局。本地按钮 height 32px 贴 toolbar 尺度，颜色/边框全走 token，不破坏视觉契约。Phase 4 plan §5.4 同理。
5. **`window.__canvasEditor` 诊断 hook** — puppeteer 脚本要读 live editor state（isGridMode / gridSize / camera z），window 暴露避免 monkey-patch。生产无副作用。
6. **snap 指示线覆盖为黑** — tldraw 默认 `hsl(0, 76%, 60%)` 饱和红，包豪斯 red 保留给 inbox/capture region（语义），canvas snap 用黑更克制（注册标尺感）。
7. **0 新依赖** — 沿用 `@tldraw/tldraw@3.15.6` + Phase 1-4 全套组件 + 全 token。
8. **mobile media query 隐藏 hint/dividers/百分比** — 让 SNAP tag 在 390px 视口下可见，但 zoom 按钮仍溢出（已知 polish 任务）。完整 mobile toolbar 重排是 Phase 5+。
