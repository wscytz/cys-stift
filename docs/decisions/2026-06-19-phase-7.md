## 关键工程决策

1. **复用 `CardService` 已有方法**:archive/unarchive/softDelete 全是 Phase 2/3 已实现;Phase 7 domain/db 零改动,纯 web 层新增(0 新依赖)。
2. **Tile + Row 双 variant 共用一个组件**:避免两套 CSS;共用蓝条/meta/选中态逻辑。
3. **多选 Set 状态不可变更新**:`new Set(prev)` 删/加;切换 selectMode / 批量操作后 `clearSelected()` 防泄漏。
4. **浮动工具条 z-index 50** < CaptureHost Mini Input 200 < Modal 100;互斥显示无冲突。
5. **时间轴日分组用 UTC ISO date**:避免本地时区偏移;P9 暴露本地时区。
6. **批量 soft-delete 不二次确认**(Lean):软删只标 `deletedAt`,DB 不真删;P9 补二次确认。
7. **Archive 不开 detail modal**:避免复制 inbox `CardDetail`(tagged Phase 3);tile onClick 留 no-op。
8. **首页 Archive 入口蓝箭头**:三色分明(红 inbox / 黑 canvas / 蓝 archive)。
