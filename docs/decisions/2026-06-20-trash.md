## 背景

软删(`softDelete` 设 `deletedAt`)后,**整个 UI 没有任何地方能看到/恢复已删卡**——inbox 与 archive 都过滤 `!deletedAt`,软删成了事实上的永久删除。但 inbox 软删弹窗文案承诺了"可恢复"(原句:`"you can recover it later from the database"`)——承诺了却没实现,是产品完整性 + 诚实性缺口。

## 决策

**入口 = 新 `/trash` 路由**,不是 archive 三 tab。三分离(inbox/archive/trash)更清晰,spec 没限定 UI 形态,选最简单。spec §4.2 的 `archived` / `deletedAt` 三态在数据层早就有,只缺视图。

## 关键工程决策

| 决策 | 选项 | 选择 | 理由 |
|---|---|---|---|
| 入口位置 | archive 三 tab / 新 /trash 路由 | **新 /trash** | 三分离(inbox 活跃 / archive 归档 / trash 已删)更清晰,spec §4.2 三态在数据层早有,只缺视图 |
| `restore` 字段 | 清 deletedAt + 改 archived / canvasPosition / **只清 deletedAt** | **只清** | 卡自然回原视图,domain 不知道"它原来在哪"——单一真相源是卡自身字段 |
| `restore`/`hardDelete` 返回 | void / **boolean** | **boolean** | 返 boolean 让调用方知道是否真改了一张卡;其他 service 方法维持原签名(零破坏) |
| Delete forever 确认 | 直接删 / Modal 二次 / **Modal 二次 + 要求打字 "delete"** | **Modal 二次** | MVP,信任 Modal 拦截;打字 "delete" 留后 |
| selectMode | 多选批量 / **单卡操作** | **单卡** | MVP 先验证核心闭环;批量 restore/hardDelete 留后(archive 批量模式可复用) |
| 视觉 | 重做 / **复用 ArchiveCardTile** | **复用** | 白底黑边 + 蓝条是通用卡,archive 恰好蓝条;不重做 |
| `region: 'trash'` 颜色 | 显式返 gray / **default 已返 gray** | **default** | `regionColorForStripe` default 返 gray,加联合类型即可,无 if 分支 |
| media gc | hardDelete 时清孤儿 / **不动** | **不动** | Phase 2.5 OPFS 时统一 gc,YAGNI |
| 定期清空 trash | 实现 / **不做** | **不做** | 未要求,YAGNI |

## 显式留后(findings 剩余 + UX 洞)

- **#4 `editor.dispose` 猴补丁** + **#5 `editor.store.listen` 无 filter** —— `canvas-editor.tsx`,留到下次动 canvas 时一起重构成 useEffect
- **批量 restore / hardDelete** —— archive 批量模式可复用
- **media gc** —— hardDelete 只删 card 记录,关联 media assets 留孤儿,Phase 2.5 OPFS 时统一 gc
- **定期自动清空 trash**(保留期)—— 未要求,YAGNI
- **批量 soft-delete 二次确认** —— archive floater 软删多卡时没确认,误删风险
- **send-to-canvas 反向动作** —— 卡上了画布后无"从画布拿回 inbox"
- **archive tile no-op** —— `archive/page.tsx:110` 注释说明;P6.5b 抽的 CardDetail 没接上归档卡
- **OPFS 真实落盘** —— media base64 撑爆 localStorage quota;Phase 2.5 独立 phase
- **Phase 8 Tauri build** —— Rust 就绪(cargo 1.96),`cargo check` 通过;待 `pnpm tauri build` + 签名公证(需 Apple 证书 $99/年)
