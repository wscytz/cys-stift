## 教训(从我此前的 reset --hard 灾难中)

1. **reset --hard 前必须先确认 stash 或 commit**。我把 v0.18/19/20/21 未 commit 的代码 reset 掉了,永久丢失。
2. **untracked 文件用 mv 不可靠**(nested directory 失败)→ 应该 `git stash --include-untracked`。
3. **subagent 跑实施任务容易 scope creep** → 用户明确要求"别开 subagent",全程手动。
4. **占位值要实测验证**(我设 `--app-menu-height: 56px` 是估算,实际 69px)→ 测量驱动。

## 修复明细

### 1. i18n lib(`apps/web/src/lib/i18n/`)
- `messages.ts`:`{ zh, en }` 扁平字典,200+ key,`as const` 联合类型
- `index.tsx`:`I18nProvider` + `useI18n()` → `{ locale, t, setLocale }`
  - `t(key, params?)` `{x}` 插值,`String(v ?? '')` 兜底
  - `setLocale` 同步写 localStorage + `<html lang>`
  - `useState(loadLocale)` 惰性初始化(防 SSR 串语言)

### 2. settingsStore 加 locale
- `Settings` 加 `locale: 'zh'|'en'`,默认 `'zh'`
- `isValid()` 校验 locale
- `updateLocale()` API

### 3. layout 接 Provider
- `<I18nProvider>` 包 `<AppMenu> {children} <CaptureHost>`,全局可访问 useI18n

### 4. AppMenu 双语 + ZH/EN 切换器
- 5 个 nav 入口用 `t('nav.*')`
- 右上角 ZH/EN 切换按钮调 `setLocale`
- 恢复 4px 灰色条(Part B1)

### 5. 各页文案迁移(主线程手做,每页一 commit)
