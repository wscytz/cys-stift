# cy's Stift — 产品简报

> 本文档供**产品路线规划与定位研究**使用。
> 只讲产品本身与设计意图,**不含技术实现细节**(不涉及具体框架、数据库、代码结构等)。
> 读这份不需要任何技术背景。
> AI 方向见 [`ai-direction.md`](ai-direction.md)。

---

## 一句话

**cy's Stift** — 本地优先的灵感画布,包豪斯风格。
**"你的灵感,在画布上生长。"**

---

## 它是什么

一个把"灵感"从闪念养养成品的桌面工具。核心是一条动线:

```
快速捕获(3 秒,随手记) → 收件箱(编辑整理) → 画布(自由地"养":挪、连、归类、变形) → 归档(沉淀)
```

关键在那个**"养"**字——想法落到画布不是终点,是起点。所以**画布不是附属视图,是产品的心脏**。这一点决定了它和市面上所有"记笔记/画白板/管知识"工具的根本差异。

---

## 它不是什么(先划清边界)

- **不是笔记软件**(Notion 式):不做大而全的文档库/数据库
- **不是白板**(Miro / tldraw 式):不是纯协作画图工具
- **不是知识库**(Obsidian 式):重心不在"管理已有知识"
- 它是**从"捕获"到"养成"到"沉淀"的一条完整动线**,画布居中

---

## 核心理念

1. **本地优先** — 数据在用户手里,不在云端。这是信念,不是可选项。
2. **形随功能(包豪斯)** — 克制与几何是设计约束,不是装饰滤镜。少即是多。
3. **数据可迁移,不锁定** — 本地数据随时能导出为开放格式(文本 / 图片 / 通用备份)。云同步若有,也只是叠加层,不是逃生口。
4. **为人类 + AI 协作而设计** — 画布不只给人看,也给 AI 读懂和修改。

---

## 差异化(为什么是它)

市面上"记笔记""画白板""管知识"的工具很多,但它们的画布**要么是纯图形(只能看),要么是封闭对象(只有它自家软件能改)**。

cy's Stift 的画布有一个独特性质(暂称**"转义"**):

> **画布上的一切,都能用一段确定的文字完整描述;反过来,这段文字也能驱动画布变化。**

这句话的产品含义是:**任何一个 AI 工具(无需专门适配、无需联网我们的服务器)都能读懂这张画布,并直接编辑它**——哪怕 AI 偶尔出错,底下的规则也能兜住不崩溃。

这让"**AI 协助你养画布**"变得廉价而自然:你可以把画布丢给任意 AI,让它帮你整理、重排、连线,再把结果接回来。没有锁定,没有门槛。

再加上**本地优先**:你的灵感数据默认不离开你的设备(除非你主动导出)。

**三点组合 = 独特卖点候选:**
1. **画布居中**(不是笔记的附属)
2. **本地优先**(数据是你的)
3. **文字可驱动画布**(任何 AI 都能协作)

---

## 今天能做什么(用户视角)

- **捕获**:全局快捷键 / 迷你输入框 / 文件拖拽——3 秒记下任何东西(文字、链接、代码、图片、文档)
- **收件箱**:多媒介编辑,草稿自动保存,一键发送到画布
- **画布**:自由排布卡片;手绘 / 图形 / 文字 / 分区框;关系连线;自动整理(思维导图 / 流程图 / 网格等多种范式);对齐与分布;模板;鸟瞰导航
- **关系网络**:卡片之间建立带语义的关系(如"引用""派生自""相关");双链自动连线;**全局图谱**视图看清全貌;智能推荐"可能相关"的卡片
- **AI 协作**:自带 API Key 即可用(支持主流云厂商,**也支持完全本地、免费、离线的模型**);能帮你自动排版 / 分组 / 找相似 / 总结 / 改写,甚至**对话式修改画布**(你提需求 → AI 提议改动 → 你确认后应用)
- **导出与备份**:图片 / Markdown / 全量数据备份(含画布),开放格式,可迁移到别的工具
- **其他**:全文搜索、标签、时间线、命令面板、暗色模式、中英双语

---

## 当前阶段(诚实表述)

- **个人项目**,核心动线已完整闭合,处于持续打磨期
- 桌面端可用(macOS,可打包为安装包),**尚未正式对外发布或分发**
- 目前**作者即唯一用户与测试者**,真实外部反馈很少
- 美学(包豪斯)与隐私(本地优先)是刻意坚持的产品立场,不是权宜之计

---

## 明确不做 / 边界

- ❌ 不做云端托管 / 不做账号体系(本地优先是信念)
- ❌ 不做"AI 看图"类视觉功能(隐私考量,暂不做)
- ❌ 暂不做移动端 / 联网多人实时协作
- ❌ 不做大而全的"第二个 Notion"
