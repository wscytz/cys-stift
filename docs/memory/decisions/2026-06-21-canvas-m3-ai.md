# 2026-06-21 · v0.29.0-canvas-m3-ai

> 来源: 用户对话(2026-06-21:加 AI 元素 — 先做 AI 访问设置 + 定制化操作 + 参考 AFFiNE 0.26.3 架构)。

## 设计

M3 给 cy's Stift 加一个**完全可选、可关闭、密钥不外泄**的 AI 层。**核心约束**:本地优先 — 没配 AI 时应用一切照旧(零按钮、零状态、零 main chunk 增量;3 个 provider 走 dynamic-import-style 注册,factory 注册 12 行,providers/index.ts 的 registerDefaultProviders 在第一次用户操作时调一次)。

### 关键决策

1. **3 provider:OpenAI / Anthropic / Ollama** —— 不做 Azure / Gemini / 本地 GGUF 直接加载(用户明确选 3 个 + Ollama 已覆盖本地场景)。每个 provider 用原生 HTTP + SSE/NDJSON,**不开 SDK**(openai/anthropic SDK 是 Node 优先,bundle +100KB+)。
2. **API key 明文 localStorage + UI 警告 banner** —— 不做 AES-GCM(本地优先,YAGNI;用户接受风险)。Warning 文案显式说明「仅本机 / 不上传 / 公共设备勿用」。**M4 可升级 OS keychain**(browser 端 `window.crypto.subtle` 可用,但需用户主动解锁)。
3. **Maker-based factory**(存 `Map<ProviderId, (cfg) => AIProvider>`)—— OpenAI/Anthropic 的 apiKey 必须闭包进 instance,不能共享。Factory = 「maker 字典」,每次 `create(cfg)` 现场 new,防止 apiKey 泄漏到跨 config 实例。
4. **AI 按钮在 `ai === null || !enabled` 时不渲染**(不是 disabled)—— 用户没配 AI 时,UI 完全干净,符合「本地优先不打扰」原则。Card-detail `cd__actions` 区只在 `useAIEnabled()` 返回 true 时挂 3 个 ✨ 按钮;canvas-toolbar 同样只在 `useAIEnabled() && selectedCardsCount >= 2` 时挂 ✨ auto-relate。
5. **3 个卡片 action + 1 个画布 action**(summarize / improveWriting / translate / autoRelate)—— 不抄 AFFiNE 的 30+(他们有云端协作语境)。`translate` 子菜单 zh↔en 二选一,`improveWriting` UI label 是 `Rewrite`(labelKey 'card.rewrite')。
6. **画布 auto-relate 复用 M2.1 `createArrowFromHandle`** —— 不重写箭头创建路径。`inferRelationTypeFromContext` 复用 M2.3 关键词匹配(命中就建箭头,没命中跳过,避免噪声)。
7. **三协议 SSE 解析策略** —— OpenAI 手写 SSE 累加(`data:` 简单);Anthropic 用 `eventsource-parser`(v1.1.2 API 是 `createParser((ev) => {...})` 直接接函数,非 v2 的 `{onEvent: ...}`);Ollama 走 NDJSON(不是 SSE,`done: true` 终止)。
8. **AI popover 内嵌浮层**而不是 Modal —— 流式输出时 Modal 体验差;inline 浮在按钮旁 + Replace / Append as new / Cancel 三选项。AbortController 在 unmount 触发,AbortError 静默吞掉。
9. **`getCurrentAI()` module-level singleton** —— `<AIProviderSync />` 在 layout 订阅 settings-store 推到 module-level;popover 异步流式渲染中 hook 重渲染易出 stale closure,同步读最直接。Card-detail 的 `useAIEnabled()` 走 hook(用于按钮条件渲染),popover 走 `getCurrentAI()`(用于读 cfg 发起 fetch)。
10. **vitest 引入** —— web 包从 0 测试到 12 测试(纯函数 / provider factory / safe-href AI 校验);jsdom env。**只测纯逻辑,不测 React 组件**(避免 testing-library 增量)。

### 不做(显式 defer)

- AI 关系类型自定义 prompt(M3 用硬编码 4 类型,用户已在 M2.3 接受关键词匹配)
- AI 自动生成全新卡片(本 plan 是「操作现有卡」,不是「造卡」)
- AI 嵌入(embedding) / 向量搜索(本地优先边界 + 当前数据量级不需要)
- 流式输出 cancel 按钮(AbortSignal 已接,AbortError 静默;M3.1 可补)
- AES-GCM 加密 API key(YAGNI;M4 可加,基于 OS keychain)
- Provider 配额监控(各家 API 自带 dashboard,UI 重复)
- 多 key 轮换 / 多模型 ensemble(单 key 足够)
- 多模态(image input 给 GPT-4V / Claude vision)—— M2.2 已经支持图片 capture,M3 不接 vision
- AI 创建箭头的概率/置信度阈值(目前「命中关键词就建」,没命中跳过)
- SSE retry / reconnect(用户主动 retry)
- Provider 配额耗尽 graceful degradation(M4)
- **Canvas modal (M2.5 export 也没接) 接入 AI 按钮** —— 计划原话「card-detail-modal 接同样 props」,但 canvas modal 没有 actions prop 概念、且 export 按钮在 canvas 也没接。强行扩会改 canvas modal 内部 button list,超出 M3 范围。**M3.1 / M4 可补**。
