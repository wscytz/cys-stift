## 修复明细

### 1. danger variant: red-fill → red-outline

**根因**(spec §3.1):
- `packages/ui/src/components/button.module.css` 旧 `.danger` = `background: var(--color-red); color: var(--color-white);`
- AppMenu Capture 按钮 = 同样红底白字
- 视觉权重相同 → 用户在 card detail Modal 看 4 个按钮时,Soft-delete 比 Archive/Send-to-canvas 都"重",但实际可恢复(从 trash 恢复),不应该比"主行动"更显眼

**修法**(spec §4.1):
- `.danger` = `background: white; color: red; border: 2px solid red`
- hover 用 `var(--color-red-soft)`(浅红洗)
- 2px border 比 default 1px 厚,仍传达"重要"语义,但不与"主行动"红填充混淆
- `padding` offset 1px 避免 2px border 让按钮变宽

**影响**:7 处 `variant="danger"`(archive × 2 / trash × 2 / canvas / mini-input / card-detail × 2 / design-page)自动同步,无业务逻辑改动。

### 2. archive tab 空文案

**修法**(`apps/web/src/app/inbox/page.tsx`):
- 标题:`'No archived cards.'` → `'Nothing archived yet.'`(对比 inbox 空态 `'No cards yet.'`,语气一致)
- 描述:`'Archived cards show up here. Unarchive to bring them back to the inbox.'` → `'Archive from the active tab to move cards here. They stay archived until you unarchive them or soft-delete from here.'`
- 新文案**指明操作入口**(active tab)和**归档 vs 软删除的区别**(spec §6.5)

### 3. SVG favicon

**根因**: `apps/web/public/` 不存在,Next.js 静态导出默认 fallback `/favicon.ico` → 404。

**修法**:新建 `apps/web/src/app/icon.svg`(Next.js App Router 自动发现 + 自动注入 `<link>`):
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" fill="#0a0a0a"/>
  <rect x="6.4" y="6.4" width="19.2" height="19.2" fill="#d40000"/>
</svg>
```

**设计**:黑底 + 60% 居中红方块 —— 6 原色品牌(spec §5.4),与 Inbox 红条、Capture 按钮同一红色 token(`#d40000` = `--color-red` light)。

**为什么不用 `app/icon.tsx`**:`ImageResponse` 走 server runtime,静态导出不支持(实测 `Failed to collect page data for /icon`)。SVG 文件 Next.js 原生支持静态导出。

**为什么有 inline hex**:`icon.svg` 是图形输出,token CSS vars 在 SVG 文件里不生效。这是 web/ 目录**唯一**写 hex 的地方。
